Exercise 3.1: Three-Column Responsive Grid

Create columns that stack vertically on mobile, two per row on tablets, three on desktops

Understanding Breakpoints
Device Size Class Prefix Example
Mobile < 576px None (default) .col-12
Tablet ≥ 576px -sm- .col-sm-6
Small Desktop ≥ 768px -md- .col-md-6
Large Desktop ≥ 992px -lg- .col-lg-4
Extra Large ≥ 1200px -xl- .col-xl-3

Key Concepts Explained
  • .container - Wraps content with responsive max-width
  • .row - Creates a horizontal group of columns
  • .col-* - Defines column width (1-12 units per row)
  • .g-3 - Adds gap/spacing between columns
  • No class on mobile = full width (12 columns)
  • .col-md-6 = 50% width starting at medium (768px)
  • .col-lg-4 = 33.33% width starting at large (992px)

Exercise 3.2: Container, Row, Column Usage

Best practices for using grid classes appropriately

Container Types
.container - Responsive fixed-width

Has max-width that changes at each breakpoint. Best for most cases.

.container-fluid - Full-width

Takes 100% width of viewport. Use for hero sections or full-screen layouts.

.container-{breakpoint} - Responsive fluid

Example: .container-lg - fluid until large breakpoint, then fixed.

Grid Structure Rules
  1. .row must always be INSIDE a .container
  2. .col must always be INSIDE a .row
  3. Never nest rows without columns
  4. A row contains up to 12 columns
  5. Column widths add up: 3 columns of .col-lg-4 = full row
Your Task
  1. Create a file exercise3-2.html
  2. Build a responsive layout with:
    • A header with .container-fluid (full width)
    • A main content area with .container
    • Multiple rows with different column configurations
    • At least 2 nested grids (grid inside a column)
  3. Test on different screen sizes
  4. Verify the layout is responsive
Bootstrap 12-Column Grid

Bootstrap divides each row into 12 equal columns:

Common widths:

  • 4 columns = 33.33% (.col-lg-4)
  • 6 columns = 50% (.col-lg-6)
  • 8 columns = 66.67% (.col-lg-8)
  • 12 columns = 100% (.col-12)
